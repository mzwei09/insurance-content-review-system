# 审核进度与用户体验优化说明

## 一、已实现的前端优化（不依赖后端改动）

### 1. 进度覆盖层与加载动画

- **全屏进度覆盖层**：审核开始时显示半透明遮罩，避免用户误操作
- **阿里云风格卡片**：橙色边框、毛玻璃背景，与现有设计保持一致
- **进度条**：渐变橙色进度条，带光泽动画，模拟审核进度
- **旋转加载图标**：复用现有 `loading-spinner` 样式

### 2. 进度与状态文案

- **主标题**：根据内容类型动态显示
  - 纯文本：`正在分析文本内容，请稍候...`
  - 纯图片：`正在分析图片内容，请稍候...`
  - 图文混合：`正在分析文本与图片内容，请稍候...`
- **副标题**：说明 AI 正在执行的操作
- **进度状态**：`正在审核第 N 张图片，共 M 张...`（基于预估时间模拟）
- **预估剩余时间**：`预计还需约 X 秒`，随进度递减

### 3. 预估时间算法

- 纯文本审核：约 5 秒
- 每张图片：约 12 秒（提取文字 + 合规审核）
- 图文混合：5 + 图片数 × 12 秒

### 4. 取消审核

- **取消按钮**：进度覆盖层底部提供「取消审核」按钮
- **实现方式**：使用 `AbortController` 取消 `fetch` 请求
- **取消后**：关闭覆盖层，恢复按钮和输入区，不弹错误提示

### 5. 防重复提交与禁用状态

- **开始审核按钮**：审核中禁用，显示 loading 状态
- **示例按钮**：审核中禁用，防止误点覆盖输入
- **上传区域**：审核中禁用（`pointer-events: none`），半透明显示

### 6. 视觉反馈

- 进度条填充带光泽动画（`progress-shine`）
- 审核中上传区添加 `reviewing` 类，降低透明度
- 按钮使用 `disabled:opacity-60` 等 Tailwind 禁用样式

## 二、后端 API 改动需求（可选，用于实时进度）

若需实现**真实逐张图片的进度反馈**，需后端支持流式或分步返回。

### 方案 A：Server-Sent Events (SSE)

**新增接口**：`GET /api/review-multimodal-stream`

- 请求：通过 query 或首包传递任务 ID，或使用 POST + 流式响应
- 响应：`text/event-stream`，每完成一张图片发送一个事件

```text
event: image_result
data: {"image_index": 1, "compliance": true, "reasoning": "..."}

event: image_result
data: {"image_index": 2, "compliance": false, "violation_types": ["夸大收益"]}

event: done
data: {"compliance": false, "image_results": [...], "text_result": {...}}
```

**前端改动**：使用 `EventSource` 或 `fetch` + `ReadableStream` 解析 SSE，实时更新进度和部分结果。

### 方案 B：轮询接口

**新增接口**：`GET /api/review-multimodal-status/{job_id}`

- 提交审核时返回 `job_id`
- 前端轮询该接口，返回已完成的图片数量和部分结果
- 全部完成后返回完整结果

### 方案 C：分步请求（每张图片单独审核）

**现有接口**：`POST /api/review-multimodal` 支持单张图片

- 前端循环调用，每次传一张图片
- 每完成一次请求，更新进度并追加结果
- 无需后端改动，但请求次数多，可能增加延迟和复杂度

## 三、当前实现与后续扩展

| 能力           | 当前实现                     | 依赖后端 |
|----------------|------------------------------|----------|
| 进度条         | 基于预估时间的模拟进度       | 否       |
| 进度文案       | 模拟「第 N 张，共 M 张」     | 否       |
| 预估剩余时间   | 基于图片数量的估算           | 否       |
| 取消审核       | AbortController 取消 fetch   | 否       |
| 防重复提交     | 按钮/上传区/示例按钮禁用     | 否       |
| 真实逐张进度   | 未实现                       | 是（SSE/轮询/分步） |

当前方案**不依赖后端改动**，即可明显改善多图审核时的等待体验。若后续接入 SSE 或轮询接口，可将模拟进度替换为真实进度。
