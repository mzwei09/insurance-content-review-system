## 🔨 使用方式

### 构建知识库

```bash
# 从这些文档构建向量知识库
python scripts/build_knowledge_base.py
```

该脚本会：
1. 解析所有文档（PDF/DOC/DOCX）
2. 提取监管条文
3. 使用百炼 Embedding API 向量化
4. 构建 FAISS 索引
5. 保存到 `data/vectorstore/`

### 添加新文档

如需添加新的监管文档：

1. 将文档放入本目录
2. 支持格式：PDF、DOC、DOCX
3. 重新运行 `python scripts/build_knowledge_base.py`

## 📋 文档要求

- **格式**: PDF、DOC 或 DOCX
- **编码**: UTF-8（DOCX）或标准 PDF 编码
- **结构**: 建议包含明确的条文编号（如"第X条"）
- **内容**: 保险行业相关的监管规定

## ⚠️ 注意事项

1. **版权**: 这些文档为公开的监管文件，仅用于合规审核
2. **更新**: 监管文档更新后，需重新构建知识库
3. **备份**: 建议保留原始文档的备份
4. **不要提交到根目录**: 所有监管文档应放在本目录下

## 🔗 相关文件

- `scripts/build_knowledge_base.py` - 知识库构建脚本
- `data/knowledge_base.json` - 解析后的条文 JSON
- `data/vectorstore/` - FAISS 向量索引
- `src/document_parser.py` - 文档解析模块
