## 🚀 人工验收步骤（5分钟）

### 步骤1：启动系统

```bash
cd <项目根目录>   # 替换为实际克隆路径，如 insurance-content-review-system
bash start.sh
```

**预期结果**：
- 看到 "[0/6] 检查端口 8000..."
- 看到 "✅ 端口 8000 可用"
- 看到 "[1/6] 检查 Python 环境..."
- 看到 "[2/6] 安装依赖..."（已安装会跳过）
- 看到 "[4/6] 初始化数据库..."
- 看到 "[5/6] 知识库已存在，跳过构建"
- 看到 "[6/6] 启动服务..."
- 看到 "服务已就绪"
- 浏览器自动打开 http://localhost:8000

**如果端口被占用**：
- 系统会提示 "❌ 错误：端口 8000 已被占用"
- 解决方案1：`lsof -ti:8000 | xargs kill -9`（停止占用进程）
- 解决方案2：`bash start.sh --port 8001`（使用其他端口）

**如果遇到架构错误**：
- 脚本会自动使用 `arch -arm64 /usr/bin/python3`
- 或者手动使用：`arch -arm64 bash start.sh`

## 🐛 问题排查

### 问题1：启动失败（架构错误）

**症状**：
```
ImportError: dlopen ... (mach-o file, but is an incompatible architecture)
```

**解决方案**：
```bash
# start.sh 应该已自动处理，如果仍失败，手动执行：
arch -arm64 /usr/bin/python3 -m pip install --upgrade pip
arch -arm64 /usr/bin/python3 -m pip install -r requirements.txt
arch -arm64 bash start.sh
```

### 问题2：端口被占用

**症状**：
```
OSError: [Errno 48] Address already in use
```

**解决方案**：
```bash
# 查找占用端口的进程
lsof -i :8000

# 杀死进程
kill -9 <PID>
```

### 问题3：API密钥验证失败

**症状**：点击"验证"后提示失败

**排查**：
- 检查API密钥是否正确（以 `sk-` 开头）
- 确认API密钥有效且有余额
- 检查网络连接
- 查看浏览器控制台（F12）错误信息
- 查看服务日志：`tail -f logs/app.log`
